# Reproduce The Public Showing

This note is about recreating the current public demonstration, not rebuilding
the whole project from scratch.

## Public route set

The current public story is centered on these routes:

- `/`
- `/shadow`
- `/portal`
- `/world`
- `/artifact`
- `/docs/`

## What each route is for

### `/`

Public entry and framing.

It should immediately show:

- what the system is
- where to click
- the public icon artifact

### `/artifact`

Portable artifact surface.

This is where you demonstrate:

- witness surface
- scanner surface
- import/export
- artifact-first sharing

### `/shadow`

Dependency field surface.

This is where you show:

- Pascal lattice structure
- wormhole movement
- projection without making layout the truth

### `/portal`

Runtime explanation surface.

This is where the runtime story becomes visible as a live view.

### `/world`

Composition surface.

This is where canonical events become world-facing composition objects.

### `/docs/`

The formal doctrine.

Use this when someone wants the exact rules and document chain.

## Minimal live walkthrough

If you only have a few minutes, use this order:

1. show `/`
2. point to the artifact card
3. open `/artifact`
4. show `/shadow`
5. show `/portal`
6. show `/world`
7. point to `/docs/` as the formal reference

## What a good public showing proves

A good showing should make the following feel true:

- the system has one core event law
- the same event can be seen in multiple surfaces
- artifacts are portable and decodable
- readable and scannable forms belong to one artifact law
- the public UI is not random; it is a set of consistent views over one system

## Last practical note

If a public page feels confusing, simplify the top of the page before adding
more features.

The public showing works best when the first thing people see is:

- one sentence
- a small number of clear choices
- one visible artifact
- one obvious path to drill deeper
